# Cloudflare AI Bot Policy

The site sits behind Cloudflare and Cloudflare's "Block AI Scrapers and Crawlers" managed feature is currently enabled. The block is injected into `robots.txt` between the `# BEGIN Cloudflare Managed content` / `# END Cloudflare Managed Content` markers — it is not editable from the Shopify theme.

## Current state (verified 2026-04-09)

`https://www.realskincare.com/robots.txt` blocks:

- `GPTBot` — OpenAI training
- `Google-Extended` — controls Google AI Overviews / Gemini eligibility
- `ClaudeBot` — Anthropic
- `Applebot-Extended` — Apple Intelligence
- `CCBot`, `Bytespider`, `Amazonbot`, `meta-externalagent` — training crawlers

Plus a site-wide `Content-Signal: search=yes,ai-train=no`.

**Search-time bots that are not blocked (citations still possible):**
- `OAI-SearchBot`, `ChatGPT-User` — ChatGPT Search
- `PerplexityBot` — Perplexity

## Decided policy

**Unblock the four search-time / citation-producing bots. Leave pure training crawlers blocked.**

| Bot | Action | Why |
|---|---|---|
| `Google-Extended` | **Allow** | Required for Google AI Overviews and Gemini citations |
| `OAI-SearchBot` | Allow (already) | ChatGPT Search citations |
| `ChatGPT-User` | Allow (already) | ChatGPT Search live fetch |
| `PerplexityBot` | Allow (already) | Perplexity citations |
| `ClaudeBot` | **Allow** | claude.ai citations |
| `GPTBot` | Block | Training crawler, no citation upside |
| `CCBot` | Block | Common Crawl, public dataset |
| `Bytespider` | Block | TikTok/ByteDance training |
| `Amazonbot` | Block | Training |
| `Applebot-Extended` | Block | Apple Intelligence training (separate from `Applebot` search) |
| `meta-externalagent` | Block | Meta training |

## How to apply

The `# BEGIN Cloudflare Managed content` block is generated by Cloudflare's managed feature, so the change must be made there — not in `robots.txt` directly.

1. Cloudflare dashboard → select the `realskincare.com` zone
2. **Security → Bots → Configure**
3. Find **AI Scrapers and Crawlers** (also called "Block AI bots")
4. Switch from **Block all AI bots** to **Allow specific** (or disable the managed block entirely if no granular option exists)
5. If granular allow-list is not available, disable the managed block and manage `robots.txt` from the Shopify theme `robots.txt.liquid` instead — explicitly disallowing the four training crawlers and allowing the rest.

## Verify after change

```
curl -s https://www.realskincare.com/robots.txt | grep -E "GPTBot|Google-Extended|ClaudeBot|PerplexityBot"
```

Expected: `Google-Extended`, `ClaudeBot`, `PerplexityBot`, `OAI-SearchBot`, `ChatGPT-User` should NOT appear in any `Disallow: /` block. `GPTBot`, `CCBot`, `Bytespider` should still appear.

## Why this matters

Without this change, the AI Citation Tracker will report ~0 citations from Gemini, AI Overviews, and claude.ai indefinitely — those engines cannot index pages they cannot crawl. ChatGPT Search and Perplexity will continue to work because their crawlers were never blocked.
